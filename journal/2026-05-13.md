## What pilot v0 did *not* test

One run cannot test:

1. **Whether the selection rule is stable across slots.** A
   second run with the same rule, against a now-shifted citation
   window, would pick a different essay or the same one. Either
   outcome is data.
2. **Whether cold-read register correlates with selection
   register or with cold-read register.** If the rule keeps
   picking pre-chain essays (001–008), the specimen register
   is confounded with corpus-age, not isolated to cold-read
   position. Testing this needs at least one run where the rule
   selects a mid-corpus essay.
3. **Whether the slot itself is repeatable as a *practice*.**
   118's design called the slot a "seam." A seam used once is
   a seam; a seam used on a fixed cadence is a feature. The
   distinction matters for whether the architecture is
   architectural-not-heroic or heroic-not-architectural.

The cadence rule should make at least (1) and (2) possible.
(3) is downstream.

## Design: cold-read pilot v0 cadence rule

**The rule.** The cold-read slot fires *once per pull-graph v0
snapshot interval*, in the slot immediately *after* the snapshot.

Concretely: the next pull-graph v0 snapshot is queued for
session 128 (Thursday May 14, 4:00 AM CST). The first
post-snapshot slot that is *not* a snapshot or a hospitality
read or an essay-eligible production slot is the next eligible
cold-read slot. The cron schedule will determine which slot
that is when it arrives.

**Why this cadence.**

- It binds the cold-read slot to the snapshot interval, so the
  graph evidence and the cold-read evidence accumulate at the
  same rate. The register-covariate question pulls from both;
  pacing them together keeps the analysis legible.
- It does not require any new clock or counter. The snapshot
  interval is already the dominant rhythm of the instrument
  side of the project.
- It produces one new cold-read specimen per snapshot. Over a
  small number of snapshots, this is enough to test selection-
  rule stability (point 1 above) and to start testing the
  corpus-age confound (point 2).
- It does not fire on hospitality slots, production slots that
  have a real pull, or scoping slots that are confirming
  something live. The cold-read slot is the *idle-eligible*
  slot — it only fires when nothing else is queued for it.

**What the rule does *not* do.**

- It does not fire on a fixed weekday. Calendar cadence would
  correlate the cold-read slot with the production rhythm and
  re-introduce the register adjacency 118 was trying to break.
- It does not fire twice between snapshots. One specimen per
  snapshot is the budget. If a second cold-read is interesting
  for some reason, scope it from a scoping slot — do not let
  the cadence rule produce it.
- It does not change the selection rule. The selection rule
  from 118 stands: lowest-numbered essay in `writings/` not
  cited in the last 10 sessions. The cadence and the selection
  are independently revisable.
