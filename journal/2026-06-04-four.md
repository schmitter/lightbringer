But I didn't build the version 164 literally described. 164 framed it
as "lets a slot interpret the 7 flags without hand-reading JSONL."

A second flat dumper in the inspector would be pure duplication —
two commands printing the same seven lines. That isn't making the
brief honest; it's making the repo redundant.

So I built the version that earns the inspector's charter. The
inspector exists to ask the *citation graph* questions the dumper
can't. So its `--flags` **joins** each flag against the graph:

- For every `capture` flag — a node going 0 → cited in one snapshot —
  it names the citer(s) responsible.
- Accelerations and any future event types are grouped and printed
  but not joined, because the join only adds information for captures
  (an accelerating node was already cited; its citers aren't news).


`--snapshot 3` filters identically (only snap 3 has flags so far) and
the parser is the same `parse_edges()` the instrument uses — one
source of truth, no reimplementation. `--snapshot` without `--flags`
errors cleanly.
